Программа для визуализации каркасной модели в трехмерном пространстве 3DViewer_v2.0.

  • разработана на языке C++ стандарта C++17;
  • соблюден Google Style;
  • разработана в соответствии с принципами объектно-ориентированного программирования, структурный подход не используется;
  • обеспечено полное покрытие unit-тестами модулей, связанных с загрузкой моделей и аффинными преобразованиями;
  • Графический пользовательский интерфейс реализован на базе Qt;
  • реализована с использованием паттерна MVC;
  • Классы реализованы внутри пространства имен s21;

Программа предоставляет следующие возможности:

  • Загружать каркасную модель из файла формата obj (поддержка только списка вершин и поверхностей);
  • Перемещать модель на заданное расстояние относительно осей X, Y, Z;
  • Поворачивать модель на заданный угол относительно своих осей X, Y, Z;
  • Масштабировать модель на заданное значение;
  • Настраивать тип проекции (параллельная и центральная);
  • Настраивать тип (сплошная, пунктирная), цвет и толщину ребер, способ отображения (отсутствует, круг, квадрат), цвет и размер вершин;
  • Выбирать цвет фона;
  • Настройки сохраняются между перезапусками программы;
  • Cохранять полученные («отрендеренные») изображения в файл в форматах bmp и jpeg;
  • По специальной кнопке записывать небольшие «скринкасты» - текущие пользовательские аффинные преобразования загруженного объекта в gif-анимацию (640x480, 10fps, 5s).

Графический пользовательский интерфейс содержит:

  • Кнопку для выбора файла с моделью и поле для вывода его названия;
  • Зону визуализации каркасной модели;
  • Кнопку/кнопки и поля ввода для перемещения модели;
  • Кнопку/кнопки и поля ввода для поворота модели;
  • Кнопку/кнопки и поля ввода для масштабирования модели;
  • Информацию о загруженной модели - название файла, кол-во вершин и ребер;

Команды make:

  • make install -- установка приложения

  • make uninstall -- удаление приложения

  • make all -- пересборка проекта + формирования отчёта о покрытии + установка приложения

  • make test -- покрытие unit-тестами модулей, связанных с загрузкой моделей и аффинными преобразованиями;
  • make gcov_report -- формирования отчёта о покрытии функций тестами
  • make dvi -- описание проекта
  • make rebuild -- пересборка проекта
  • make clean -- очистка проекта
  • make clean_all -- очистка проекта + удаление приложения
  • make style -- проверка файлов проекта на Google Style
  • make correct_style -- приведения файлов проекта к Google Style
  • make dist -- архивирование проекта
